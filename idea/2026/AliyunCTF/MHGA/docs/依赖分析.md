## Hessian 白名单分析

### 默认允许的包

Hessian 4.0.66 默认允许以下包的反序列化：

```java
// 基础 Java 类
java.lang.*
java.util.*
java.io.*
java.math.*
java.net.*
java.security.*
java.text.*
java.sql.*

// 其他白名单类 (需要验证)
javax.management.*
// ...
```

### 默认**不**在白名单中

- `org.apache.commons.*` - CommonsCollections 不直接可用
- `com.fasterxml.jackson.*` - Jackson gadget 受限
- `com.google.guava.*` - Guava gadget 受限
- `com.rometools.*` - ROME 不在白名单

## 攻击链选择

### 可用攻击向量

```
┌─────────────────────────────────────────────────────────────┐
│                    攻击向量优先级                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. HessianProxyFactory + 恶意 Hessian 服务器               │
│     └── 使用白名单内接口 + JDK 原生 gadget                   │
│                                                             │
│  2. Jackson Typing gadget (如果白名单允许)                  │
│     └── jackson-databind 2.16.0 已知漏洞                    │
│                                                             │
│  3. 结合 JNDI + Hessian                                     │
│     └── 本题主要攻击路径                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### JDK 11 原生 Gadget

考虑以下 JDK 原生利用链：

1. **Javax.management 相关**
   - `MLet` - 可能的类加载
   - `JMX` 相关类

2. **LDAP 相关**
   - `Reference` - 已用于攻击
   - `DirContext` - 可能的 gadget

3. **其他 JDK gadget**
   - 需要进一步研究 JDK 11 特有链

## 版本特定漏洞

### Jackson 2.16.0

Jackson-databind 2.16.0 存在 CVE-2023-35116 等漏洞，但可能受 Hessian 白名单限制。

### Log4j 2.20.0

Log4j 2.20.0 已经修复了 Log4Shell (CVE-2021-44228)，但可能存在其他漏洞。

### Guava 30.1.1

Guava 历史上存在反序列化问题，但需要具体分析。

## 结论

这个 jar 包包含了丰富的攻击面，但 Hessian 4.0.66 的白名单机制是主要障碍。建议使用 **JNDI + HessianProxyFactory** 的攻击路径，通过恶意 Hessian 服务器返回白名单允许的类来绕过限制。
